# Проект интернет-магазина на Django

## Описание проекта

Это учебный проект интернет-магазина, разработанный для изучения работы с моделями, шаблонами и формами в Django. Проект демонстрирует использование моделей Django ORM, шаблонов, фильтров, тегов, наследования шаблонов, работы со статическими файлами и административной панелью.

## Структура проекта

```
myshop/
├── myshop/              # Основная директория проекта
│   ├── settings.py     # Настройки проекта
│   ├── urls.py         # Главный файл URL-маршрутов
│   └── ...
├── shop/               # Приложение интернет-магазина
│   ├── models.py      # Модель Product
│   ├── views.py       # Представления product_list и product_detail
│   ├── urls.py        # URL-маршруты приложения
│   └── admin.py       # Настройка административной панели
├── templates/          # Глобальные шаблоны
│   └── shop/
│       ├── base.html           # Базовый шаблон
│       ├── product_list.html   # Шаблон списка товаров
│       └── product_detail.html # Шаблон детальной страницы товара
├── static/             # Статические файлы
│   └── shop/
│       └── css/
│           └── styles.css  # Стили для оформления
└── media/              # Медиа-файлы (изображения товаров)
    └── products/
```

## Установка и запуск

1. **Установите зависимости:**
   ```bash
   pip install django pillow
   ```

2. **Примените миграции:**
   ```bash
   python manage.py makemigrations
   python manage.py migrate
   ```

3. **Создайте суперпользователя (опционально):**
   ```bash
   python manage.py createsuperuser
   ```

4. **Запустите сервер разработки:**
   ```bash
   python manage.py runserver
   ```

5. **Откройте браузер:**
   - Главная страница: http://127.0.0.1:8000/
   - Админ-панель: http://127.0.0.1:8000/admin/

## Основные возможности

### Модель Product
- **name** - название товара (CharField)
- **description** - описание товара (TextField)
- **price** - цена товара (DecimalField с валидацией)
- **image** - изображение товара (ImageField, опционально)
- **created_at** - дата создания (DateTimeField, автоматически)

### Представления

1. **product_list** - отображает список всех товаров
   - URL: `/`
   - Использует шаблон `product_list.html`
   - Передает в контекст список всех товаров

2. **product_detail** - отображает детальную информацию о товаре
   - URL: `/product/<product_id>/`
   - Использует шаблон `product_detail.html`
   - Обрабатывает случай, когда товар не найден (404)

### Шаблоны

1. **base.html** - базовый шаблон с хэдером и футером
   - Использует наследование блоков
   - Подключает статические CSS файлы
   - Содержит навигационное меню

2. **product_list.html** - список товаров
   - Наследуется от `base.html`
   - Использует цикл `{% for %}` для отображения товаров
   - Применяет фильтры: `length`, `floatformat`, `truncatechars`, `date`
   - Обрабатывает случай пустого каталога

3. **product_detail.html** - детальная страница товара
   - Наследуется от `base.html`
   - Отображает полную информацию о товаре
   - Обрабатывает ошибку 404, если товар не найден

### Статические файлы

- CSS файлы находятся в `static/shop/css/styles.css`
- Используется тег `{% static %}` для подключения файлов
- Адаптивный дизайн для мобильных устройств

## Использованные технологии и концепции

- **Шаблоны Django**: создание и рендеринг HTML-шаблонов
- **Наследование шаблонов**: использование `{% extends %}` и `{% block %}`
- **Шаблонные теги**: `{% for %}`, `{% if %}`, `{% url %}`, `{% static %}`
- **Фильтры**: `floatformat`, `truncatechars`, `date`, `length`, `linebreaks`
- **Статические файлы**: подключение CSS через `{% static %}`
- **Медиа-файлы**: работа с изображениями через `ImageField`
- **Обработка ошибок**: использование `get_object_or_404` и обработка 404 в шаблонах

## Дополнительные функции

- Обработка отсутствующих изображений
