# Решение проблемы недоступности админ-панели

## Возможные причины и решения

### 1. Сервер Django не запущен

**Проблема:** Страница недоступна, потому что сервер не работает.

**Решение:**
```bash
python manage.py runserver
```

Вы должны увидеть:
```
Starting development server at http://127.0.0.1:8000/
Quit the server with CTRL-BREAK.
```

**Важно:** Сервер должен быть запущен в отдельном терминале и продолжать работать.

---

### 2. Не создан суперпользователь

**Проблема:** Вы не можете войти в админ-панель без учётных данных.

**Решение:**
```bash
python manage.py createsuperuser
```

Введите:
- Email: (например, admin@example.com)
- Password: (придумайте пароль)
- Password (again): (повторите пароль)

---

### 3. Неправильный URL

**Правильные URL:**
- Главная админка: http://127.0.0.1:8000/admin/
- Издательства: http://127.0.0.1:8000/admin/books/publisher/
- Магазины: http://127.0.0.1:8000/admin/books/store/
- Книги: http://127.0.0.1:8000/admin/books/book/
- Отзывы: http://127.0.0.1:8000/admin/books/review/

**Важно:** 
- Используйте `http://` (не `https://`)
- Используйте `127.0.0.1` или `localhost`
- Порт должен быть `8000` (если не изменён)

---

### 4. Приложение не добавлено в INSTALLED_APPS

**Проверка:**
Откройте файл `myshop/settings.py` и убедитесь, что есть строка:
```python
INSTALLED_APPS = [
    ...
    'books',  # Приложение для работы с книгами (Тема 20)
]
```

---

### 5. Миграции не применены

**Проверка и решение:**
```bash
python manage.py migrate books
```

---

## Пошаговая проверка

### Шаг 1: Проверьте, что сервер запущен

Откройте новый терминал и выполните:
```bash
python manage.py runserver
```

Оставьте этот терминал открытым. Сервер должен работать.

### Шаг 2: Откройте браузер

В браузере откройте:
```
http://127.0.0.1:8000/admin/
```

### Шаг 3: Войдите в систему

Если видите форму входа:
- Введите email суперпользователя
- Введите пароль
- Нажмите "Войти"

Если суперпользователя нет, создайте его (см. выше).

### Шаг 4: Проверьте раздел BOOKS

После входа вы должны увидеть раздел "BOOKS" с моделями:
- Books
- Publishers
- Reviews
- Stores

---

## Быстрая диагностика

Выполните эти команды по порядку:

```bash
# 1. Проверка конфигурации
python manage.py check

# 2. Применение миграций
python manage.py migrate

# 3. Создание суперпользователя (если нужно)
python manage.py createsuperuser

# 4. Запуск сервера
python manage.py runserver
```

---

## Если ничего не помогает

1. Проверьте, что вы в правильной директории проекта
2. Убедитесь, что виртуальное окружение активировано (если используется)
3. Проверьте, нет ли ошибок в терминале где запущен сервер
4. Попробуйте перезапустить сервер (Ctrl+C, затем снова `python manage.py runserver`)
